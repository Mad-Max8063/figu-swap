# Antigravity Developer Skill: Auto-Backup & Farewell Routine

Esta skill contiene las directrices para despedir de manera robusta y segura cada sesión de pair-programming con el usuario, asegurando que todos los archivos nuevos y modificados queden respaldados en GitHub automáticamente.

## Cuándo usar esta Skill
Activar esta skill en cuanto el usuario indique que se va a retirar, a dormir, o al despedirse al final de una sesión de trabajo.

## Pasos del Procedimiento de Despedida

1. **Revisión del Estado del Repositorio**:
   - Ejecutar `git status` en la raíz del proyecto para evaluar qué archivos han sido creados o modificados.

2. **Crear Commit Informativo**:
   - Hacer un staging completo de las modificaciones válidas utilizando `git add .`.
   - Realizar un commit semántico con un mensaje descriptivo y profesional que resuma las tareas completadas en la sesión (ej. `git commit -m "feat: setup Firebase hosting configurations and migrate Express backend into Cloud Functions"`).

3. **Subir los Cambios (Git Push)**:
   - Enviar las actualizaciones al repositorio remoto ejecutando `git push origin [nombre-de-la-rama]`.

4. **Resumen de Cierre Conciso**:
   - Proveer un reporte factual de los logros de la sesión, los archivos guardados en GitHub, y dejar asentado exactamente en qué punto o tarea pendiente se retomará el trabajo en la próxima sesión.
   - Desear un buen descanso o jornada al usuario de forma cordial pero profesional.
